# features Rules

- Deep module boundary: import other modules only through their public `index.ts` or `index.tsx` entrypoints.
- Frontend module files must carry restoration tier comments when they encode P1/P2 evidence.
- P0 may record manifest facts. P1 may encode IPC wrappers, command names, argument keys, and page-module boundaries. P2 may add source skeletons only.
- Do not present minified bundle analysis as recovered original source.
- Do not add business behavior without evidence.
- Do not write external reference project names in code, comments, docs, or commits.
- Do not restore deleted promotional assets or chat-invite material, private host, private path, machine name, or username content.

Scope: feature public APIs and private feature implementation.
