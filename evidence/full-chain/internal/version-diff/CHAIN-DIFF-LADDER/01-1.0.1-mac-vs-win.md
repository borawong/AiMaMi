## 6. hop 01 诚实小结

- **变更类型统一=平台分支差异**：同版本（1.0.1）跨平台实现分叉，非版本演进。
- **API 调用确定、函数名推定**：无 PDB，所有 Rust 名 `inferred_fn`，但 WinAPI 调用 + 控制流直接读 pseudocode（certain）。
- **Win call-tree 无 WinAPI 叶**：Ghidra 不列 IAT thunk 为 callee 边——WinAPI 在 pseudocode 正文 + `target-discovery.txt` 锚定，本 hop API 叶映射以 IAT 锚点为权威。
- **Mac 对照非逐行 pseudocode**：基于符号全名+地址+strings（Mac platform leaf 未单独反编译）；标注为符号级对照。
- **autostart Run-field inferred**：PE 有 `dev.aimami.auto-switch` 标识 + Reg imports，但无字面 `…\Run` 串，标 inferred。
- **跨模块共享 helper 如实标注**：FUN_140ddd5d0(Reg reader, proxy+tray 共用)、FUN_14061b220/b9c0(proxy callee 但 autostart 也调)——manifest 按 string-anchor 归 proxy，标 cross-module。FUN_1400db030/da970 是通用析构（非 proxy callee），正确归 unclassified。
- **48 runtime+unclassified 不强归模块**：共享 CRT/alloc/std/UTF/panic，强塞模块=不诚实。
- **Win 仅 1.0.1 有 artifact**：1.0.4/1.0.5/1.0.6 的 Win 面 Unknown，不从 Mac bundle 推断（见各包 SYSTEM-DIFF）。
