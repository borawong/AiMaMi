## 🎯 产品定位

AiMaMi 不是聊天工具，也不是模型入口。  
它解决的是 Codex 的使用管理问题：

- 账号多，切换频繁
- 额度变化需要持续关注
- 自动切换需要可控而不是“静默发生”
- 本地会话、MCP、Skills、配置文件需要统一维护

## ✨ 核心能力

### 👤 账号管理

- 查看本地账号列表、活跃账号与基础信息
- 一键切换账号，并应用到 Codex 当前运行环境
- 支持添加、导入、删除、复制等高频操作

### 📊 当前态总览

- 集中展示当前账号、5 小时额度、1 周额度
- 同步展示账号数、会话数、MCP 数、Skills 数
- 内置本地环境健康检查，快速定位异常

### 🔁 自动切换

- 支持按 5 小时 / 1 周额度阈值检测候选账号
- 达到阈值后前置主窗口并提醒确认
- 用户可选择暂不切换，或立即重启并切换

### 🧠 多入口状态一致

- 主窗口、刘海、托盘围绕统一当前态刷新
- 适合高频观察，不必始终停留在主窗口

### 💬 会话管理

- 浏览、搜索、删除本地会话
- 快速了解会话规模与活跃情况

### 🔌 MCP 管理

- 管理本地 MCP 服务配置
- 支持新增、编辑、启停、删除

### 🧩 Skills 管理

- 查看已安装 Skills
- 支持导入、移除、备份、恢复

### 🛠️ 维护工具

- 系统诊断
- 数据清理
- 注册表重建
- Codex 重启

## 🚀 典型使用场景

### 日常使用

- 打开主窗口查看当前账号与额度
- 不打开主窗口时，通过刘海或托盘快速感知状态

### 额度临界

- 当前账号接近阈值
- 系统给出候选账号
- 确认后完成切换并让新账号立即生效

### 本地整理

- 统一整理账号、会话、MCP、Skills
- 本地环境异常时，直接使用维护工具排查与恢复

## 💡 适合谁

- 长期使用 Codex 的个人用户
- 同时维护多个账号的人
- 需要稳定观察额度并减少手工切换的人
- 希望把本地 Codex 环境管理做得更清晰的人

## 📌 价值

AiMaMi 的重点不是增加更多入口，而是把最重要的事情收拢到一起：

- 当前是谁在工作
- 当前额度还剩多少
- 是否该切账号
- 本地环境是否健康

当这些问题能被稳定地放在一个桌面工具里，Codex 的日常使用成本会明显下降。
